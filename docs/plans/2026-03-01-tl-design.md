## 1. Overview

**tl** is a Rust-based native Mac/Linux TUI application for rapid triage of Windows forensic collections. It provides a unified, timeline-first view of all forensic artifacts extracted from a system, with vi-key navigation and sub-30-second time-to-first-view.

### Problem Statement

Velociraptor targeted collections (and similar triage tools like KAPE, CyLR) extract forensic artifacts into directory-based archives rather than full disk images. Standard forensic tools (OSForensics, X-Ways, Autopsy, FTK) expect unified E01/raw disk images and cannot process these collections natively.

Specifically, Velociraptor collections split a single volume across two path-encoding schemes:
- `uploads/ntfs/%5C%5C.%5CC%3A/` (decoded: `\\.\C:\`) — raw NTFS artifacts via the NTFS accessor
- `uploads/auto/C%3A/` (decoded: `C:\`) — file-level artifacts via the auto accessor

No existing tool merges these into a coherent analysis view.

### Solution

Parse artifacts directly from the collection format (zip/directory), normalize paths, correlate across artifact types, and present a unified timeline in an interactive TUI with vi keybindings.

### Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Name | `tl` | Short, memorable |
| Language | Rust | Speed, safety, cross-platform, excellent forensic crate ecosystem |
| UI | ratatui + crossterm TUI, vi keys | Fast, keyboard-driven, no GUI dependencies |
| Primary view | Timeline-first, merged single row per entity | Most natural for triage workflow |
| Collection scope | Universal (Velociraptor first, then KAPE, CyLR, raw dirs) | Abstraction layer enables this |
| Speed target | <30s to browsable timeline for 2GB collection | Parse MFT + small artifacts upfront, stream EVTX in background |
| Export | CSV (bodyfile) + JSON/JSONL + HTML report | Maximum interoperability |

### Anti-Goals

- NOT a full disk image tool (no E01/raw/VMDK support)
- NOT a replacement for X-Ways/Autopsy for deep analysis
- NOT a GUI application
- NOT a Windows application (Mac/Linux only)

### 2.1 Collection Abstraction Layer

Each collection format is handled by a provider implementing `CollectionProvider`:

```rust
trait CollectionProvider: Send + Sync {
    fn discover(&self) -> ArtifactManifest;
    fn open(&self, path: &NormalizedPath) -> io::Result<Box<dyn Read + Send>>;
    fn source_path(&self, path: &NormalizedPath) -> WindowsPath;
    fn metadata(&self) -> CollectionMetadata;
}
```

**VelociraptorProvider**: Reads from zip (streaming, no full extraction). Handles dual path encoding: URL-decodes `%5C%5C.%5CC%3A` and `C%3A`, merges both into unified `C:\`. Tags files with accessor type for provenance.

**KapeProvider**: Maps KAPE's `C/` directory structure to normalized paths.

**RawProvider**: For loose artifact directories.

### 2.2 Artifact Discovery

On collection open, the provider scans available files and produces an `ArtifactManifest` — a categorized inventory of what's available:

```rust
struct ArtifactManifest {
    ntfs_core: NtfsCoreArtifacts,      // $MFT, $UsnJrnl, $LogFile, etc.
    registry_hives: Vec<RegistryHive>,  // per-user + system hives
    event_logs: Vec<EventLogFile>,      // all .evtx files
    prefetch: Vec<PrefetchFile>,        // all .pf files
    lnk_files: Vec<LnkFile>,
    jump_lists: Vec<JumpListFile>,
    scheduled_tasks: Vec<TaskFile>,
    recycle_bin: Vec<RecycleBinEntry>,
    amcache: Option<AmcacheHive>,
    wmi_repository: Option<WmiRepo>,
    bits_db: Option<BitsDb>,
    other: Vec<GenericArtifact>,
    windows_installations: Vec<WindowsInstall>, // Windows/ + Windows.old/
}
```

### 2.3 Parallel Parser Orchestrator

Uses `rayon` for parallel parsing. Strategy:

1. **Phase 1 (blocking)**: Parse $MFT (single-threaded, sequential read). Build MFT index.
2. **Phase 2 (parallel)**: Parse all small artifacts concurrently (registry, LNK, Jump Lists, Prefetch, Shimcache, Amcache, Recycle Bin, Scheduled Tasks).
3. **Phase 3 (background streaming)**: Parse event logs and $UsnJrnl asynchronously. Update timeline as records arrive. TUI shows progress indicator.

This achieves the <30s target: MFT index + small artifacts are fast; EVTX can stream in after.

## 3. Timeline Model

### 3.1 Timeline Entry

```rust
struct TimelineEntry {
    entity_id: EntityId,
    path: WindowsPath,
    timestamps: TimestampSet,
    event_type: EventType,
    sources: SmallVec<[ArtifactSource; 4]>,
    anomalies: AnomalyFlags,
    metadata: EntryMetadata,
}

enum EventType {
    FileCreate,
    FileModify,
    FileAccess,
    FileDelete,
    FileRename { old_name: String },
    Execute,
    RegistryModify,
    ServiceInstall,
    ScheduledTaskCreate,
    UserLogon,
    UserLogoff,
    ProcessCreate,
    NetworkConnection,
    BitsTransfer,
    RdpSession,
    Other(String),
}

struct TimestampSet {
    // $MFT $STANDARD_INFORMATION
    si_created: Option<WinTimestamp>,
    si_modified: Option<WinTimestamp>,
    si_accessed: Option<WinTimestamp>,
    si_entry_modified: Option<WinTimestamp>,
    // $MFT $FILE_NAME
    fn_created: Option<WinTimestamp>,
    fn_modified: Option<WinTimestamp>,
    fn_accessed: Option<WinTimestamp>,
    fn_entry_modified: Option<WinTimestamp>,
    // Other artifact sources
    usn_timestamp: Option<WinTimestamp>,
    lnk_target_created: Option<WinTimestamp>,
    lnk_target_modified: Option<WinTimestamp>,
    lnk_target_accessed: Option<WinTimestamp>,
    jumplist_timestamp: Option<WinTimestamp>,
    prefetch_last_run: Option<Vec<WinTimestamp>>,
    amcache_timestamp: Option<WinTimestamp>,
    shimcache_timestamp: Option<WinTimestamp>,
    evtx_timestamp: Option<WinTimestamp>,
}

bitflags! {
    struct AnomalyFlags: u32 {
        const TIMESTOMPED_SI_LT_FN     = 0b0000_0001;
        const TIMESTOMPED_ZERO_NANOS   = 0b0000_0010;
        const METADATA_BACKDATED       = 0b0000_0100;
        const NO_USN_CREATE            = 0b0000_1000;
        const LOG_GAP_DETECTED         = 0b0001_0000;
        const LOG_CLEARED              = 0b0010_0000;
        const EXECUTION_NO_PREFETCH    = 0b0100_0000;
        const HIDDEN_ADS               = 0b1000_0000;
    }
}
```

### 3.2 Entity Resolution

Merges rows from different artifact parsers that refer to the same entity:

1. **MFT entry number**: Direct match when artifact references MFT record number
2. **Normalized path**: Case-insensitive Windows path comparison
3. **Hash**: SHA256/MD5 from Amcache, Prefetch file references
4. **USN rename tracking**: Follow RENAME_OLD_NAME/RENAME_NEW_NAME pairs

The unified timeline is stored in a `BTreeMap<WinTimestamp, Vec<TimelineEntry>>` for ordered traversal, with a secondary index by entity_id for entity-centric views.

### 3.3 Anomaly Detection

During entity merge, the following checks run:

| Check | Condition | Flag |
|-------|-----------|------|
| Timestomping (basic) | SI Created < FN Created | TIMESTOMPED_SI_LT_FN |
| Timestomping (tool) | SI timestamps have 0 nanoseconds | TIMESTOMPED_ZERO_NANOS |
| Metadata backdating | SI Created > embedded document CreateDate | METADATA_BACKDATED |
| Missing USN create | File exists in MFT but no FILE_CREATE in USN | NO_USN_CREATE |
| Log clearing | Event ID 1102 in Security.evtx | LOG_CLEARED |
| Log gap | >24h gap in expected continuous log | LOG_GAP_DETECTED |
| Execution anomaly | Amcache/Shimcache entry with no Prefetch | EXECUTION_NO_PREFETCH |
| Hidden data | MFT shows ADS count > 0 | HIDDEN_ADS |

## 4. Artifact Coverage

### Tier 1: NTFS Core (Timeline Backbone)

| Artifact | Source | Parser | Forensic Value |
|----------|--------|--------|----------------|
| $MFT | ntfs accessor | `mft` crate | Every file record, SI+FN timestamps, ADS, resident data |
| $UsnJrnl:$J | ntfs accessor | Custom | File creates/deletes/renames/modifications with timestamps |
| $LogFile | ntfs accessor | Custom | NTFS transaction-level operations |
| $MFTMirr | ntfs accessor | `mft` crate | Integrity check of first 4 MFT records |
| $Secure:$SDS | ntfs accessor | Custom | File ownership and ACLs |
| $Boot | ntfs accessor | Custom | Volume serial number, cluster size |

### Tier 2: Execution Evidence

| Artifact | Path | Parser | Forensic Value |
|----------|------|--------|----------------|
| Prefetch (.pf) | Windows/Prefetch/ | `frnsc-prefetch` | Last 8 run times, run count, referenced files/dirs |
| Amcache.hve | Windows/appcompat/Programs/ | `frnsc-amcache` | SHA1, publisher, install date, first execution |
| Shimcache | SYSTEM hive AppCompatCache | Custom (via `notatin`) | Program existence in insertion order |
| BAM/DAM | SYSTEM hive bam/ | Custom (via `notatin`) | Last execution time per user SID |
| UserAssist | NTUSER.DAT UserAssist | Custom (via `notatin`) | GUI program run count, focus time, last run |

### Tier 3: Persistence & Autoruns

| Artifact | Source | Parser | Forensic Value |
|----------|--------|--------|----------------|
| Run/RunOnce keys | NTUSER.DAT + SOFTWARE | `notatin` | Auto-start programs |
| Services | SYSTEM hive | `notatin` | Service configurations, ImagePath |
| Scheduled Tasks | System32/Tasks/ (XML) | `serde` + custom | Task definitions with triggers/actions |
| WMI Subscriptions | WMI Repository OBJECTS.DATA | Custom | WMI persistence (event consumers) |
| Startup LNKs | Start Menu/Startup/ | `lnk` crate | Startup shortcut targets |

### Tier 4: User Activity

| Artifact | Source | Parser | Forensic Value |
|----------|--------|--------|----------------|
| LNK files | Recent/, Start Menu/, etc. | `lnk` crate | Target path, timestamps, volume serial, MAC addr |
| Jump Lists (Auto) | AutomaticDestinations/ | `jumplist_parser` | Recent files per application |
| Jump Lists (Custom) | CustomDestinations/ | `jumplist_parser` | Pinned items per application |
| Shellbags | UsrClass.dat | Custom (via `notatin`) | Folder browsing history with timestamps |
| MRU lists | NTUSER.DAT | Custom (via `notatin`) | Recent documents, Open/Save dialogs |
| Recycle Bin | $Recycle.Bin/$I* | Custom | Deleted file original path + deletion time |

### Tier 5: Event Logs

| Log Channel | Key Event IDs | Forensic Value |
|-------------|---------------|----------------|
| Security | 4624/4625 (logon), 4648 (explicit creds), 4672 (privs), 4688 (process), 4720 (user create), 1102 (cleared) | Auth, execution, clearing |
| System | 7034-7036 (services), 7045 (new service), 6005/6006 (startup/shutdown) | Services, uptime |
| PowerShell/Operational | 4103 (module), 4104 (script block) | Script execution |
| TaskScheduler/Operational | 106/140/141 (task lifecycle), 200/201 (execution) | Scheduled task activity |
| TerminalServices-LSM/Op | 21-25 (RDP session lifecycle) | Remote access |
| TerminalServices-RDP | 1149 (network RDP) | RDP source IP |
| BITS-Client/Operational | 3/4/59-61 (BITS jobs) | Download activity |
| Defender/Operational | 1006-1009 (detections), 5001 (disabled) | AV events |
| Sysmon/Operational | 1/3/7/11/13/22 (if present) | Rich telemetry |

### Tier 6: Network & Lateral Movement

| Artifact | Source | Parser | Forensic Value |
|----------|--------|--------|----------------|
| BITS database | qmgr.db (ESE) | `ese_parser` | Background download jobs |
| RDP history | TerminalServices ETL + evtx | `evtx` + custom | Session history |
| Network profiles | SOFTWARE hive | `notatin` | Connected networks with dates |
| Firewall rules | Windows Firewall evtx | `evtx` | Connection events |

### Tier 7: Additional Artifacts

| Artifact | Source | Parser | Forensic Value |
|----------|--------|--------|----------------|
| setupapi.dev.log | Windows/INF/ | Text parser | USB device install history |
| Windows Search DB | Windows.db (SQLite) | `rusqlite` | Indexed file metadata |
| WER Reports | WER/ (text files) | Text parser | Crash data with timestamps |
| Defender logs | MPLog, MPDetection | Text parser | Scan/detection activity |

## 7. Phased Implementation Plan

### Phase 1: MVP -- NTFS Timeline Core
- Cargo project scaffold (`tl` binary)
- Collection abstraction trait + VelociraptorProvider (zip, path normalization)
- $MFT parser integration: SI + FN timestamps, all file records
- Timeline model: BTreeMap, basic TimelineEntry
- TUI: ratatui table with vi keys (j/k, /, gg/G, Ctrl-d/u)
- Timestomping detection (SI vs FN)
- CSV bodyfile export

### Phase 2: USN Journal + File Operations
- Custom $UsnJrnl:$J parser (USN_RECORD_V2/V3)
- Entity resolution: merge USN events with MFT entries by MFT ref + path
- USN reason codes -> EventType mapping
- TUI: time range filter, detail pane with USN sequence

### Phase 3: Execution Evidence
- Prefetch parser (frnsc-prefetch): run times, run count, file refs
- Amcache parser (frnsc-amcache): SHA1, publisher, install date
- Shimcache parser (custom, from SYSTEM hive via notatin)
- BAM/DAM parser (custom, from SYSTEM hive)
- Execution event type, TUI `e` filter
- Cross-reference: created -> ran correlation

### Phase 4: User Activity Artifacts
- LNK parser (lnk crate): target timestamps, volume info
- Jump List parser (jumplist_parser): AutoDest + CustomDest
- UserAssist parser (custom via notatin)
- MRU list parser (custom via notatin)
- Shellbag parser (custom via notatin)
- Recycle Bin $I parser (custom)
- Entity resolution: merge LNK/JumpList into MFT entries

### Phase 5: Registry Deep Dive
- Run/RunOnce keys
- Services enumeration
- Network profiles with timestamps
- USB device history (SYSTEM hive + setupapi.dev.log)
- Timezone, ComputerName, last shutdown
- Persistence summary view

### Phase 6: Event Logs
- EVTX parser (evtx crate), parallel across all log files
- Priority channel extraction with Event ID mapping
- Merge into unified timeline
- Background streaming (async parse, progressive TUI updates)
- Source filter toggles per channel

### Phase 7: Advanced Artifacts & Anti-Forensics
- $LogFile parser (custom)
- WMI Repository parser
- BITS database parser (ese_parser)
- Scheduled Tasks XML parser
- Full anomaly detection engine
- Log gap / log clearing detection

### Phase 8: Universal Triage & Export
- KapeProvider
- RawProvider
- JSON/JSONL export
- HTML report generator (standalone)
- Multi-collection diff

### Phase 9: Polish & Performance
- Memory-mapped I/O for large artifacts
- Virtualized scrolling for 100K+ rows
- Session save/restore, bookmarks
- Config file, color themes
- Headless mode (--headless)
- Man page, docs

## 8. Sample Velociraptor Collection Structure

From `Collection-A380_localdomain-2025-08-10T03_41_20Z.zip` (2963 files, ~2.35GB):

```
uploads/
  ntfs/%5C%5C.%5CC%3A/         # \\.\C:\ (NTFS accessor)
    $MFT                        # 1.03 GB
    $LogFile                    # 67 MB
    $Extend/$UsnJrnl:$J         # 36 MB
    $Extend/$UsnJrnl:$Max       # 32 bytes
    $Boot                       # 8 KB
    $Secure:$SDS                # 5 MB
  auto/C%3A/                    # C:\ (auto accessor)
    Users/
      4n6h4x0r/
        NTUSER.DAT              # 9.9 MB
        AppData/Local/Microsoft/Windows/UsrClass.dat  # 4.6 MB
        AppData/Roaming/Microsoft/Windows/Recent/
          AutomaticDestinations/  # 27 files
          CustomDestinations/     # 3 files
          *.lnk                   # ~100 files
    Windows/                     # Current OS installation
      System32/config/
        SYSTEM                   # 13 MB
        SOFTWARE                 # 107 MB
        SAM                      # 131 KB
        SECURITY                 # 32 KB
      System32/winevt/Logs/      # ~100 .evtx files
      appcompat/Programs/Amcache.hve  # 3.3 MB
      Prefetch/                  # ~584 .pf files
    Windows.old/                 # Previous OS installation
      WINDOWS/System32/config/   # Duplicate hives
      WINDOWS/Prefetch/          # Duplicate prefetch
      WINDOWS/appcompat/Programs/Amcache.hve
    $Recycle.Bin/                # Recycle bin entries
    ProgramData/Microsoft/Network/Downloader/qmgr.db  # BITS
```

Key challenge: `ntfs/` and `auto/` paths refer to the same C: volume but with different URL encodings. The VelociraptorProvider normalizes both to `C:\`.
